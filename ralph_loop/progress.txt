## Iteration 5 - US-005: Add chart placeholder instructions to agent prompts
- What was implemented: Added CHART PLACEHOLDERS instructions to 3 agent prompts so LLMs know to output `{{CHART:id}}` placeholders inline
- Files changed: `agents/finance_qa_agent.py`, `agents/dcf_agent.py`, `agents/equity_analyst_agent.py`
- Approach for each:
  - `finance_qa_agent.py`: Appended `_chart_instructions` regular string to `intro` (f-string) before assigning to `system_message`
  - `dcf_agent.py`: Inserted new CHART PLACEHOLDERS section in the plain template string (before OUTPUT FORMAT)
  - `equity_analyst_agent.py`: Concatenated a regular string to the f-string using `+ (...)` after the closing `"""`
- Learnings for future iterations:
  - Double-brace escaping applies at TWO levels: (1) Python f-string: `{{` → `{`, (2) LangChain PromptTemplate/ChatPromptTemplate `.format()`: `{{` → `{`
  - To get `{{CHART:id}}` in the LLM's rendered prompt: need `{{{{CHART:id}}}}` in the raw string that goes through ONE level of format (plain string + PromptTemplate, or regular string appended to already-evaluated f-string + ChatPromptTemplate)
  - Best practice: use a regular (non-f-string) for chart instructions and append it to the evaluated f-string. Avoids double-escaping confusion.
  - Verified with `str.format(input='x', agent_scratchpad='', ...)` — confirmed `{{CHART:id}}` appears in output
