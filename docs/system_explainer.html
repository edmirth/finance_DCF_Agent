How the debate mechanics work
01

Data snapshot first

Before any agent starts, a data collector fetches everything once — financials, prices, 11 Tavily searches, SEC filings — all in parallel. Every agent then reads the exact same numbers. No drift, no inconsistency.

02

Sequential peer context

Agents run one at a time, in order. When agent 3 starts, it can read the full findings of agents 1 and 2. This means each agent can agree, push back, or calibrate its confidence based on what its peers already said.

03

3-step reasoning per agent

Data-CoT: just describe what the numbers say. Concept-CoT: apply the framework and reach a view. Q&A: if confidence <70%, ask another agent a direct question.

04

PM scores consensus

After each round, the PM computes: alignment ratio × average confidence. Above 70% = debate over. Below 70% and under the 2-round cap = debate continues for another round.

05

Conflicts are flagged

When two agents strongly disagree, the PM flags the conflict explicitly. The next round's agents see these flagged conflicts and must address them. Unresolved conflicts surface in the final memo.

06

Final memo

The PM synthesises everything into a structured investment memo: verdict, conviction level, bull case, bear case, what would change the view, and a summary of the debate. All grounded in real data.
